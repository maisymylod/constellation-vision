# constellation-vision

[![CI](https://github.com/maisymylod/constellation-vision/actions/workflows/ci.yml/badge.svg)](https://github.com/maisymylod/constellation-vision/actions/workflows/ci.yml)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

The vision plane of the Heliosnet ground system. A compact UNet, **trained from
scratch** (no pretrained weights, no model download), does semantic
**segmentation of surface defects** on optical frames of orbital-hardware panels:
`crack`, `hot_spot`, `delamination`, `missing_fastener`, against a `background`.
The trained model is exported to **ONNX** and wired back into the
[`argus`](https://github.com/maisymylod/argus) sibling as an enrichment step.

Everything runs from a clean clone with one command and no external data. Every
metric below comes from actually training the model and evaluating it on a
held-out split (see [What's real vs simulated](#whats-real-vs-simulated)).

## Quickstart

```bash
make install-train   # CPU torch + onnx + onnxruntime, plus dev tools
make train           # train from scratch, export ONNX, evaluate, write EVAL.md
make enrich          # run the argus enrichment step on a held-out frame
```

Reproduce the headline from a clean clone without torch (runs the committed
`segmenter.onnx` through onnxruntime):

```bash
make install-eval    # onnx + onnxruntime + dev tools, no torch
make eval            # evaluate the committed segmenter.onnx on the held-out split
```

No-ML-stack path (data synthesis, metrics, enrichment postprocessing, tests):

```bash
make install         # numpy + pytest only, no torch
make data            # regenerate the deterministic train/test frame splits
make test            # tests + quality gate (metric calibration + coverage floor)
make drift           # confirm the synthetic distribution matches committed stats
```

## Live metrics (held-out split, regenerated by `make eval`)

Measured by evaluating the committed `segmenter.onnx` on 160 held-out frames.
Reproduce from a clean clone with `make eval` (onnxruntime, no torch), or retrain
from scratch and re-export with `make train`; both write these into
[`artifacts/EVAL.md`](artifacts/EVAL.md) and report identical numbers.

| Metric | Value |
|---|---|
| **Mean IoU (defect classes)** | **0.8821** |
| Pixel accuracy | 0.9921 |

| Class | IoU |
|---|---|
| background | 0.9946 |
| crack | 0.9164 |
| hot_spot | 0.7579 |
| delamination | 0.9024 |
| missing_fastener | 0.9515 |

Train frames: 480, held-out test frames: 160, frame size: 96x96, model
parameters: ~117k, run: seed 42 / 16 epochs / batch 16 / lr 1e-3.

The metric is **calibrated**, not asserted: a perfect mask scores mean IoU 1.0
and a background-only mask scores 0.0 on the defect classes. Those two anchors
are enforced as tests in CI (`tests/test_metrics.py`), so the headline measures
segmentation quality rather than bookkeeping.

## Architecture

```
data/generate.py            model/unet.py            eval/harness.py
(synthetic panel frames  -> (UNet-lite, from      -> (held-out split,
 + ground-truth masks)       scratch, ~117k params)   mean IoU + per-class IoU)
       |                          |                          |
   schema.py  (defect taxonomy: the one source of per-pixel ground truth)
       |                          |                          |
       v                          v                          v
  drift.py                 model/export_onnx.py       eval/report.py
 (distribution guard)      (segmenter.onnx)          (EVAL.md + history.jsonl)
                                  |
                                  v
                        enrich/argus_enrich.py
                 (ONNX session -> per-pixel mask -> per-defect summary)
```

A frame is built from a panel surface (illumination gradient, faint fastener
grid, sensor noise) plus zero to three injected defects, each painted into both
the image and the mask. The model never sees the painting rules; it learns the
spatial signature from pixels and is graded on masks it never saw. See
[docs/architecture.md](docs/architecture.md).

## The argus enrichment step

`argus` explores public imagery and runs ONNX inference to turn masks into
downstream detections; its built-in segmenter is a deliberately simple
deterministic threshold model, and its model card says to swap in a real trained
network. constellation-vision **is** that network for orbital-hardware frames.
The enrichment module loads `segmenter.onnx` with the same onnxruntime session
contract `argus` uses and returns a structured per-defect summary:

```bash
$ make enrich
{
  "defects_found": 3,
  "defect_area_fraction": 0.1118,
  "defects": [
    {"defect": "delamination", "pixel_count": 903, "area_fraction": 0.098, ...},
    {"defect": "crack",        "pixel_count": 114, "area_fraction": 0.0124, ...},
    {"defect": "missing_fastener", "pixel_count": 13, ...}
  ]
}
```

A test asserts the ONNX argmax agrees with the torch model on the same frame, so
the exported artifact is the model that was evaluated.

## What's real vs simulated

- **Real:** the from-scratch training, the held-out mean IoU and per-class IoU,
  the ONNX export, and the enrichment. Every published number is regenerated by
  `make eval` from a clean clone (it evaluates the committed `segmenter.onnx`), or
  from scratch by `make train`. The committed ONNX model is the artifact `make eval`
  reads; the torch `.pt` checkpoint is regenerated by `make train` and not committed.
- **Simulated:** the imagery. Frames are procedurally rendered, not photographs,
  which keeps the repo self-contained, license-clean, offline, and CI-friendly.
  No proprietary or restricted imagery is fetched, stored, or required.
- **Honest limit:** a model trained on synthetic frames does not transfer to real
  imagery without retraining; `missing_fastener` is a tiny, hyperparameter-
  sensitive class that only works because of the inverse-frequency class
  weighting. See [docs/assumptions.md](docs/assumptions.md).

## Continuous improvement

- **CI** (`.github/workflows/ci.yml`): a core job (tests + metric-calibration
  gate + coverage floor, no ML stack) and a train job (install CPU torch, train
  from scratch, enforce the mean-IoU floor, regenerate the eval artifacts, run
  the ONNX enrichment smoke test).
- **Nightly improvement loop** (`.github/workflows/improve.yml`): regenerate
  data, drift-check it, retrain, evaluate, and commit the new eval + checkpoint
  metadata **only if mean IoU genuinely improved**, with a commit message stating
  the real delta, else commit nothing. Runs under the `heliosnet-ci[bot]`
  identity, never an empty or cosmetic commit.
- **Drift check** (`make drift`): the synthetic distribution statistics are
  committed and reproduced from a clean clone; a mismatch stops the loop rather
  than retraining on silently shifted data.
- **History:** each eval appends to `artifacts/history.jsonl` so the
  metric-over-time story stays backed by reproducible numbers.

## Capabilities demonstrated

From-scratch semantic segmentation (a compact UNet in PyTorch, no pretrained
weights); programmatic, deterministic dataset synthesis with a single source of
per-pixel ground truth and a leak-free held-out split; class-imbalance handling
(inverse-frequency weighting plus Dice loss); a calibrated, regenerable
evaluation (mean IoU, per-class IoU, confusion matrix) wired to an enforced
quality gate; ONNX export and CPU inference serving; and a CI-driven
continuous-improvement loop with a synthetic-distribution drift guard.

## Layout

```
src/constellation_vision/
  schema.py              defect taxonomy (per-pixel ground-truth classes)
  config.py              paths, seeds, shapes, hyperparameters, the IoU floor
  data/generate.py       deterministic synthetic frame + mask generator
  data/drift.py          synthetic-distribution drift guard
  model/unet.py          from-scratch UNet-lite
  model/export_onnx.py   ONNX export (frame -> logits)
  train.py               train from scratch, export, evaluate, enforce the floor
  eval/metrics.py        IoU / per-class IoU / confusion matrix (pure numpy)
  eval/harness.py        run the model over the held-out split
  eval/report.py         EVAL.md + metrics.json + history.jsonl
  enrich/argus_enrich.py ONNX segmentation -> per-defect summary (argus step)
configs/                 default + documented-larger-run settings
artifacts/               EVAL.md, data_stats.json (committed); metrics.json (regen)
```
